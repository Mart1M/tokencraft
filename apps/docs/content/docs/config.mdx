title: Configuration
description: tokencraft.config.json reference
---

Place `tokencraft.config.json` at the root of your token project to control which files TokenCraft loads and how they are grouped.

## Simple file list

```json
{
  "version": 1,
  "files": [
    "tokens/core.tokens.json",
    "tokens/semantic.tokens.json"
  ]
}
```

Only listed files are loaded. Paths are relative to the workspace root.

## Named collections

Group files under collection names shown in the sidebar:

```json
{
  "version": 1,
  "collections": [
    {
      "name": "Core",
      "files": ["tokens/core.tokens.json"]
    },
    {
      "name": "Semantic",
      "files": ["tokens/semantic.tokens.json"],
      "modes": ["light", "dark"]
    }
  ]
}
```

### Modes per collection

The optional `modes` array defines mode columns for every file in that collection. If omitted, TokenCraft infers modes from the token data.

## Legacy `sources` format

TokenCraft also accepts a `sources` array with `path` entries, for compatibility with other tools:

```json
{
  "version": 1,
  "sources": [
    { "path": "tokens/core.tokens.json" }
  ]
}
```

## Auto-discovery

Without a config file, TokenCraft discovers all `**/*.tokens.json` files under the workspace root.
