title: Configuration
description: tokencraft.config.json reference
---

Place `tokencraft.config.json` at the root of your token project to control which files TokenCraft loads and how they are grouped.

## Simple file list

```json
{
  "version": 1,
  "files": [
    "tokens/core.tokens.json",
    "tokens/semantic.tokens.json"
  ]
}
```

Only listed files are loaded. Paths are relative to the workspace root.

## Named collections

Group files under collection names shown in the sidebar:

```json
{
  "version": 1,
  "collections": [
    {
      "name": "Core",
      "files": ["tokens/core.tokens.json"]
    },
    {
      "name": "Semantic",
      "files": ["tokens/semantic.tokens.json"],
      "modes": ["light", "dark"]
    }
  ]
}
```

### Modes per collection

The optional `modes` array defines mode columns for every file in that collection. If omitted, TokenCraft infers modes from the token data.

## Mode storage

Control how mode values are written on disk with the top-level `modeStorage` field:

```json
{
  "version": 1,
  "modeStorage": "value-map",
  "collections": [
    {
      "name": "Semantic",
      "files": ["tokens/semantic.tokens.json"],
      "modes": ["light", "dark"]
    }
  ]
}
```

| Value | Behavior |
| --- | --- |
| `value-map` (default) | Modes live in the **same file** as keys under `$value` / `value` |
| `separate-files` | Each mode is its **own file**; `files[i]` maps to `modes[i]` |

Choose this when opening a folder for the first time (Create workspace dialog). An existing `tokencraft.config.json` keeps its current setting.

### Separate files example

```json
{
  "version": 1,
  "modeStorage": "separate-files",
  "collections": [
    {
      "name": "Semantic",
      "files": [
        "tokens/semantic/light.tokens.json",
        "tokens/semantic/dark.tokens.json"
      ],
      "modes": ["light", "dark"]
    }
  ]
}
```

Each mode file stores scalar token values (not mode maps). TokenCraft merges them into one collection in the UI.

### Auto-detection

With `modeStorage: "separate-files"`, TokenCraft also detects sibling files that share enough token paths (for example `semantic/legacy/light.json` + `semantic/legacy/dark.json`) and merges them into one collection even when `collections` is not listed in the config. Mode names come from the file basenames; the collection name comes from the parent folder.

## Legacy `sources` format

TokenCraft also accepts a `sources` array with `path` entries, for compatibility with other tools:

```json
{
  "version": 1,
  "sources": [
    { "path": "tokens/core.tokens.json" }
  ]
}
```

## Auto-discovery

Without a config file, TokenCraft discovers all `**/*.tokens.json` files under the workspace root.
